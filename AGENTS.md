# AGENTS.md

## Команды проверки

```bash
python -m py_compile bench.py
```

## Структура проекта

- `bench.py` — запускалка. На прогон поднимает N параллельных копий, каждая со своим
  `opencode serve` на отдельном порту в своей подпапке. Подкоманда `serve` поднимает
  локальный тестовый веб-сервер из базы.
- `data/main.db` — **единственный источник правды**: отчёты прогонов, библиотека
  заданий, цены, правила бесплатности, кэш OpenRouter. Коммитится в git. JSON-файлов
  с данными на диске нет.
- `opencode.json` — **только описание агента** `coder` (системный промпт + разрешения).
- `src/` — исходники запускалки (зарезервировано).
- `data/result/` — рабочие папки агента: `<project>/<provider>_<model>/<timestamp>_<N>/`;
  для агента подпапка копии — корень мира. Подробный лог копии — в `run.log` внутри неё.
  (Сами отчёты теперь в базе, не в `report.json`.)

## Откуда берутся настройки

- **Провайдеры** (ollama, openrouter, opencode zen, …) — из `~/.config/opencode/opencode.json`.
- **API-ключи** — из `~/.local/share/opencode/auth.json` (наполняется через `opencode auth login`).
- **Модель и разрешения агента** — из проектного `opencode.json` (этот файл).

При изменении провайдеров правь **глобальный** конфиг, а не проектный.

## Запуск

```bash
# прогон бенчмарка
python bench.py --project <имя> [-p provider] [-m model] [-n копий] "<задача>"

# локальный тестовый веб-сервер (данные из data/main.db)
python bench.py serve [--port 8000]
```

Флаг `--project` обязателен. По умолчанию запускается 5 параллельных копий (`-n`). Провайдер и модель передаются раздельно (`-p`/`-m`).
